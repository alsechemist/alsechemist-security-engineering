Configuration of ModSecurity Audit Log

To enable effective log analysis and correlation in Wazuh, ModSecurity must be configured to generate structured audit logs in JSON format. JSON-formatted logs allow Wazuh to reliably parse events using the built-in JSON_Decoder, enabling precise field extraction and rule-based detection.

This section describes how to configure ModSecurity v3 to log audit events in JSON format and ensure the logs are properly written to disk for collection by the Wazuh agent.

Enabling Audit Logging

Ensure that ModSecurity audit logging is enabled by setting the appropriate directives in the ModSecurity configuration file.

To enable it the set the below Parameter:

SecAuditEngine On

When SecAuditEngine is set to On, ModSecurity generates an audit log entry for every request and response processed by the web server. This mode provides complete visibility into web traffic.

Configuring JSON Audit Log Format

ModSecurity v3 supports JSON-based audit logging using the serial audit log mechanism.

Configure the audit log type and format as follows:

SecAuditLogType Serial
SecAuditLogFormat JSON
SecAuditLog /path/to/modsec_audit.log

Where:

  • SecAuditLogType Serial writes all audit events to a single log file.

  • SecAuditLogFormat JSON enables JSON-formatted audit entries.

  • SecAuditLog defines the audit log file path.

Each audit event will be written as a single JSON object, making it suitable for ingestion by Wazuh and other log processing tools.

Verifying JSON Log Output

After applying the configuration, restart the web server and updated ModSecurity configurations will be applied immediately.

Example:

sudo systemctl restart [apache2/nginx]

Generate a test request e.g http://[server-ip]/?exec=/bin/bash that triggers a ModSecurity rule, then verify the audit log output:

tail -f /path/to/modsec_audit.log | jq
